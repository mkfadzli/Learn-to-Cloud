# Topik 1: Kawalan Versi

> ⏱️ **Anggaran masa:** 3-5 hari

Di L2C kami menggunakan Git dan GitHub untuk kawalan versi. Saya perlukan anda meluangkan beberapa hari untuk menguasai mereka.

---

## 📚 Laluan Pembelajaran

1. **Belajar:** [Pengenalan kepada Git](https://learn.microsoft.com/training/modules/intro-to-git/)

2. **Buat [akaun GitHub](https://docs.github.com/get-started/start-your-journey/creating-an-account-on-github)**

3. **Bintangkan [Repositori Learn to Cloud](https://github.com/learntocloud/learn-to-cloud)**

4. **Sediakan pembantu pembelajaran AI**

   Di L2C kami sangat menggemari kemampuan GenAI untuk membantu pembelajaran. Kami banyak menggunakannya untuk meneroka dengan lebih mendalam dan menjelaskan konsep. Ini adalah masa yang sesuai untuk anda mendaftar akaun dengan mana-mana alat yang anda suka dan mula menggunakannya untuk pembelajaran anda sendiri.

   Cuba dahulu dengan menguji pengetahuan anda tentang apa itu kawalan versi. Gunakan prompt ini sebagai templat:

    ```txt
    I'm studying cloud engineering and recently learned about version control. I will provide you an explanation about it and please ask me any questions if my explanation is not clear. I want to make sure I really understand this concept so please do not correct me, simple ask questions until I get the explanation right. Here is my explanation: version control is
    ```

5. **[Pasang VS Code](https://code.visualstudio.com/)** - Kami menggunakan VS Code untuk semua pengaturcaraan kami

6. **Pengguna Windows sahaja:** [Sediakan WSL](https://learn.microsoft.com/windows/wsl/install)

   Semua makmal dan alat kami menjangkakan persekitaran berasaskan Linux, WSL akan menyediakannya. Kemudian ikuti: [Get started using Visual Studio Code with Windows Subsystem for Linux](https://learn.microsoft.com/en-us/windows/wsl/tutorials/wsl-vscode)

7. **Belajari asas baris perintah:** [Command Line for Beginners – How to Use the Terminal Like a Pro](https://www.freecodecamp.org/news/command-line-for-beginners/#heading-most-common-and-useful-commands-to-use)

8. **Buat ruang kerja anda**

   Menggunakan Terminal, buat folder pada komputer anda untuk menyimpan semua makmal dan projek. Namakannya `l2c`. Gunakan perintah `mkdir` dan `cd` untuk ini.

9. **Klon repositori makmal**

   Di dalam folder `l2c` anda, jalankan: `git clone https://github.com/learntocloud/linux-ctfs`

10. **Belajari Markdown:** [Communicate using markdown](https://learn.microsoft.com/en-us/training/modules/communicate-using-markdown/)

11. **Buat README profil GitHub anda:** [Adding a profile README](https://docs.github.com/en/account-and-profile/setting-up-and-managing-your-github-profile/customizing-your-profile/managing-your-profile-readme#adding-a-profile-readme)

12. **Berlatih aliran kerja Git**

    Klon repositori README anda ke mesin tempatan. Gunakan VS Code untuk mengeditnya dan push perubahan kembali ke GitHub dari terminal. Berikut adalah [panduan hebat](https://www.freecodecamp.org/news/how-to-use-markdown-in-vscode/) tentang menggunakan markdown dengan VS Code. Tidak perlu membuat README yang sangat mewah — tulis penerangan ringkas tentang diri anda dan tambah pautan ke media sosial anda.
