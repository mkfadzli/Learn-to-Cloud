# Topik 4: Akses Jarak Jauh Yang Selamat

> ⏱️ **Anggaran masa:** 1-2 hari

Dalam topik sebelumnya, anda telah mempelajari tentang asas rangkaian cloud, termasuk VPC, subnet, dan aliran trafik. Kini, fokus adalah untuk memahami mekanisme akses jarak jauh yang selamat yang melindungi sumber cloud daripada akses tanpa kebenaran.

---

## 📚 Laluan Pembelajaran

- Kaji

  - Pengurusan akses berasaskan sesi: Menggunakan perkhidmatan native cloud seperti [AWS Systems Manager Session Manager](https://docs.aws.amazon.com/systems-manager/latest/userguide/session-manager.html), [Azure Bastion](https://learn.microsoft.com/en-us/azure/bastion/), dan [GCP IAP](https://www.youtube.com/watch?v=Vfxf2LdsXfs)

---

## 🧪 Uji Pengetahuan Anda

Setelah selesai dengan sumber yang disediakan, lihat ini [di sini](https://awstip.com/aws-dont-ssh-into-your-public-bastion-use-ssm-session-manager-instead-a5b527f8db41)
Setelah selesai mengkaji, uji pemahaman anda dengan prompt ini:

- Apakah kaedah akses jarak jauh yang selamat dalam persekitaran cloud?
- Apakah kelebihan menggunakan alat akses berasaskan sesi (contohnya, AWS SSM, Azure Bastion, atau GCP IAP) berbanding SSH/RDP langsung?

---

## 🛠️ Tugas Hands-on

- Konfigurasi akses berasaskan sesi untuk pelayan jarak jauh menggunakan AWS SSM, Azure Bastion, atau GCP IAP
- Gunakan dasar IAM untuk menguatkuasakan akses keistimewaan terkecil bagi pengguna jarak jauh

---

## ✅ Senarai Semak Topik

Sebelum meneruskan, pastikan anda telah:

- [ ] Memahami konsep pengurusan akses berasaskan sesi
- [ ] Mempelajari tentang AWS SSM, Azure Bastion, dan GCP IAP
- [ ] Memahami kelebihan keselamatan berbanding SSH/RDP langsung
- [ ] Mengkonfigurasi akses berasaskan sesi pada penyedia cloud anda
- [ ] Menggunakan dasar IAM keistimewaan terkecil untuk akses jarak jauh
