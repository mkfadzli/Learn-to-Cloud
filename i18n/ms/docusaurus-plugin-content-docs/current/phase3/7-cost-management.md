# Topik 7: Pengebilan Cloud dan Pengurusan Kos

> ⏱️ **Anggaran masa:** 2-3 hari

Perkhidmatan cloud dicas berdasarkan penggunaan. Setiap VM, pangkalan data, dan panggilan API mempunyai kos. Dalam topik ini, anda akan mempelajari cara memantau perbelanjaan, menyediakan amaran kos, dan menganggarkan perbelanjaan sebelum deploy infrastruktur. Kemahiran ini menghalang bil yang tidak dijangka dan mengajar anda membuat keputusan seni bina yang peka kos.

---

## 📚 Laluan Pembelajaran

### Konsep Teras

1. **Harga bayar-mengikut-penggunaan**: Anda dicas untuk apa yang anda gunakan, bukan bayaran di hadapan
2. **Jenis sumber mempunyai kos berbeza**: VM, storan, pemindahan data semuanya berharga berbeza
3. **Harga berbeza mengikut rantau**: Sumber yang sama mempunyai kos berbeza di rantau berbeza
4. **Kitaran pengebilan**: Biasanya bulanan. Caj muncul dalam bil anda 24-48 jam selepas penggunaan

## 🧪 Uji Pengetahuan Anda

Setelah mengkaji sumber-sumber tersebut, uji pemahaman anda dengan prompt ini:

1. Bil cloud anda lebih tinggi daripada yang dijangkakan. Bagaimanakah anda akan menyiasat sumber mana yang menyebabkan peningkatan kos?
2. Anda sedang deploy aplikasi yang akan melayani 1,000 pengguna. Bagaimanakah anda akan menganggarkan kos bulanan?
3. Bagaimanakah anda akan menyediakan amaran belanjawan untuk memberitahu anda jika perbelanjaan melebihi $50/bulan?
4. Bagaimanakah anda akan menentukan berapa kos untuk menjalankan VM 24/7 selama satu bulan?
5. Mengapa penting untuk memahami sumber apa yang sedang berjalan dalam akaun cloud anda?
6. Bagaimanakah anda akan mengoptimumkan kos untuk persekitaran pembangunan berbanding persekitaran pengeluaran?

---

## 🛠️ Tugas Hands-on

Sediakan pemantauan kos untuk infrastruktur cloud anda:

- Semak caj bulan semasa anda dalam papan pemuka pengebilan
- Kenal pasti peluang penjimatan kos dalam perbelanjaan semasa anda
- Sediakan amaran belanjawan sebanyak $50/bulan (atau had pilihan anda)

## ✅ Senarai Semak Topik

Sebelum meneruskan, pastikan anda telah:
- [ ] Memahami bagaimana pengebilan cloud dan harga berasaskan penggunaan berfungsi
- [ ] Mengkaji struktur pengebilan penyedia cloud anda
- [ ] Log masuk ke papan pemuka pengebilan anda dan menyemak kos semasa
- [ ] Mencipta belanjawan dan menyediakan amaran kos
- [ ] Menggunakan kalkulator harga untuk menganggarkan kos
- [ ] Mengkaji strategi pengoptimuman kos (menghentikan sumber yang tidak digunakan, right-sizing instance)
