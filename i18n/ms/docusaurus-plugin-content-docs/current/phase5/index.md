# Fasa 5: Keselamatan Aplikasi Cloud Anda

> ⏱️ **Anggaran masa:** 3-4 minggu
>
> Penulis: [Ethan Troy](https://hackidle.com)

Selamat datang ke Fasa 5! Fasa ini memfokuskan pada keselamatan aplikasi cloud dan infrastruktur yang telah anda bina sepanjang perjalanan anda. Anda akan mengambil aplikasi Journal API dari fasa-fasa sebelumnya dan melaksanakan kawalan keselamatan tahap perusahaan, pemantauan, dan keupayaan tindak balas insiden.

---

## 🎯 Objektif

Di akhir fasa ini, anda akan:

- Mengamankan aplikasi Journal API dengan kawalan IAM yang betul
- Melaksanakan perlindungan data dan pengurusan rahsia
- Mengkonfigurasi keselamatan rangkaian dan penyambungan yang selamat
- Menyediakan sistem pemantauan dan pemberitahuan keselamatan
- Membina pengesanan ancaman automatik dan tindak balas insiden
- Menyiapkan projek pelaksanaan keselamatan yang komprehensif

---

## 📋 Prasyarat

- Telah menyelesaikan Fasa 3: Asas Platform Cloud (telah menggunakan infrastruktur)
- Telah menyelesaikan Fasa 4: Asas DevOps (disyorkan tetapi tidak wajib)
- Akses kepada aplikasi Journal API dan infrastruktur dari fasa-fasa sebelumnya
- Akaun platform cloud (AWS, Azure, atau GCP) dengan kebenaran pentadbiran

---

## 🔗 Bagaimana Fasa Ini Membina Perjalanan Anda

Fasa ini mengambil asas praktikal yang telah anda bina dalam Fasa 1-4 dan menambah lapisan keselamatan yang menjadikan aplikasi anda sedia untuk pengeluaran:

| Fasa | Bagaimana Ia Berhubungan |
|-------|-----------------|
| **Fasa 1-2** | Gunakan kemahiran Linux dan pengaturcaraan anda untuk mengkonfigurasi alatan keselamatan |
| **Fasa 3** | Amankan sumber cloud dan rangkaian yang telah anda gunakan |
| **Fasa 4** | Tambah pengimbasan keselamatan dan pemantauan ke talian paip CI/CD anda |
| **Fasa 5** | Jadikan semuanya sedia untuk pengeluaran dengan kawalan keselamatan tahap perusahaan |

:::tip Keselamatan bukan sesuatu yang berasingan
Keselamatan bukan disiplin yang berasingan — ia adalah kemahiran penting yang meningkatkan setiap aspek kejuruteraan cloud.
:::

Untuk asas yang sangat baik dalam pemikiran keselamatan cloud, baca gambaran keselamatan pembekal ini:
- [AWS Security Pillar - Well Architected Framework](https://docs.aws.amazon.com/wellarchitected/latest/security-pillar/welcome.html)
- [Microsoft Azure - Secure](https://learn.microsoft.com/en-us/azure/architecture/framework/security/overview)
- [Google Cloud - Security Foundations](https://cloud.google.com/architecture/security-foundations)
